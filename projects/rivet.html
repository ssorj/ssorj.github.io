Rivet

Rivet is a tool for installing and testing ActiveMQ and Qpid source modules.

Using Rivet

To see what Rivet thinks it's going to do, run the \17 command.

[9

This will tell you what the module names are and what parameters each will use to do its work.

All commands (except \18) take a list of module names or \19. The latter tells Rivet to perform the operation for every modules reported by \20.

You can specify a module revision by appending \21 to the module name, as in \22. Otherwise, Rivet uses the current tip of the master or trunk branch.

]0

Use \23 to check out module source code, \24 to compile it, and \25 to deploy it under the install prefix. The \26 command implies \27, which in turn implies \28.

]1

The \29 command will not re-export a source revision it already has. Use \30 to start from scratch.

]2

The \31 command creates release artifacts. Use the \32 option to control the names of the resulting archives and their internal root directory. The default tag is the branch name, a date stamp, and a revision.

]3

The \33 command completes all of the foregoing and runs tests against it.

]4

The \34 command produces a shell-compatible environment for accessing the installed resources. You can source this output directly into your shell to update the search paths for your session.

]5

For developers, the main attraction is getting the stuff you want to develop against installed and accessible to the shell where you are working.

]6

If your environment is already setup to use an alternate install prefix, you can tell rivet to install there.
